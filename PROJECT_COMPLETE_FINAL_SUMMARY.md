## ðŸ’¡ RECOMMENDED TEACHING STRATEGIES

### **For K-2 (Picture-Based, No Coding)**

**Focus:** Concrete manipulation, pattern recognition, problem-solving foundations

**Best Practices:**
- Short sessions (10-15 minutes)
- Hands-on unplugged activities first
- Digital activities as reinforcement
- Celebrate effort and progress
- Use familiar contexts (animals, food, home, school)
- Read aloud all instructions
- Allow unlimited retries
- Focus on concepts, not terminology

**Sample Lesson Structure:**
1. Unplugged introduction (5 min): Physical sequencing with cards
2. Digital practice (7 min): Picture-based drag-drop activity
3. Reflection (3 min): "What did we learn?" discussion

### **For Grade 3 (Critical Gateway Year)**

**Focus:** Block coding fundamentals - events, loops, conditionals, variables, debugging

**Best Practices:**
- Allocate 40-50% of time to 5 gateway skills
- Ensure mastery before advancing
- Use multiple modalities (visual, kinesthetic, verbal)
- Frequent formative assessment
- Small group instruction for struggling students
- Celebrate successful debugging
- Connect coding to real-world applications

**Sample Lesson Structure:**
1. Concept introduction (10 min): What are variables? Why do we need them?
2. Guided practice (15 min): Teacher demonstrates, students follow
3. Independent practice (15 min): Students complete 2-3 skills
4. Debugging challenge (10 min): Find and fix errors
5. Reflection (5 min): Exit ticket or quick discussion

### **For Grades 4-8 (Application & Mastery)**

**Focus:** Complex projects integrating multiple concepts

**Best Practices:**
- Project-based learning (2-4 week projects)
- Student choice (pathway selection based on interests)
- Peer collaboration
- Code reviews and debugging partnerships
- Portfolio development
- Competition preparation (optional)
- Real-world connections
- Ethical considerations integrated throughout

**Sample Project Structure:**
1. Project introduction & planning (1-2 sessions)
2. Skill building (2-4 sessions): Practice required skills
3. Project development (6-10 sessions): Build with teacher support
4. Testing & debugging (2-3 sessions): Fix issues, get feedback
5. Presentation & reflection (1-2 sessions): Share and learn
