# Complete T30-T36 Analysis: Systems & Society Domain Capstone

## Overview

The Systems & Society domain (T30-T36) represents the final piece of the K-8 Computer Science skill map, adding **systems thinking, social responsibility, and ethical integration** to the technical foundation built in earlier domains.

**Completion Status**: ALL 7 TOPICS FULLY ANALYZED
- 221 total skills

## Topic Summaries with Key Dependencies

### T30: Devices, Hardware, Physical Computing & Software (32 skills)

**What Students Learn**:
- How devices work (input, processing, output)
- Distinction between hardware and software
- How programs run on different devices
- Physical computing basics (sensors, actuators)

**Grade Progression**:
- **G1**: Hardware components (display, keyboard, mouse)
- **G2**: Software tools and selection
- **G3**: Hardware-software relationships
- **G4**: Programs on different devices
- **G5**: Physical input and sensors
- **G6**: Device capabilities and limitations
- **G7**: Complex systems integration
- **G8**: Complete ecosystem understanding

**Key Dependencies**:
- Within-topic: Sequential (G1→G2→...→G8)
- **T06** (programming): Light - how programs run on devices (G2+)
- **T33** (platforms): Medium - platforms run on hardware (G5+)
- **T35** (impacts): Light - implications of device ubiquity (G7+)

**Capstone Skills (4)**:
- T30.G8.01-04: Understanding complete hardware-software ecosystems

---

### T31: Internet, Cloud & Networked Systems (31 skills)

**What Students Learn**:
- How the internet works (messages, routing, destinations)
- Cloud storage and services
- Multiplayer connectivity
- Network security and privacy

**Grade Progression**:
- **G1**: Internet as connection mechanism
- **G2**: Cloud storage concepts
- **G3**: Multiplayer games and shared experiences
- **G4**: Packets and data transmission
- **G5**: Cloud services and online tools
- **G6**: Network topologies and infrastructure
- **G7**: Internet scalability and growth
- **G8**: Internet as global system

**Key Dependencies**:
- Within-topic: Sequential
- **T06** (programming): Light - if hands-on networking projects (G2+)
- **T19** (multiplayer): Medium - multiplayer demonstrates networking (G3)
- **T33** (cloud APIs): Medium - cloud services as platform (G6+)
- **T25** (data): Light - cloud data storage (G5-6)

**Capstone Skills (4)**:
- T31.G8.01-04: Understanding internet infrastructure and implications

---

### T32: Cybersecurity & Safe Online Behavior (34 skills)

**What Students Learn**:
- Personal digital security (passwords, privacy)
- Recognizing threats and scams
- Security mechanisms (encryption, authentication)
- Ethical security practices
- **CRITICAL ETHICS BRIDGE**: Security as personal AND collective responsibility

**Grade Progression**:
- **G1-2**: Personal security awareness
  - Passwords and what not to share
  - Privacy and personal information
  - Distinguishing real from online
- **G3-4**: Threat recognition
  - Scams and phishing
  - Malware and viruses
  - Safe online practices
- **G5-6**: Security mechanisms
  - Encryption basics
  - Authentication systems
  - Secure passwords
- **G7-8**: Advanced & ethical
  - Complex security concepts
  - Responsible security practices
  - **ETHICS**: Security as shared responsibility

**Key Dependencies**:
- Within-topic: Sequential (STRONG - security builds)
- **T09** (variables): Light - passwords as secure variables (G2-3)
- **T31** (internet): Medium - network security (G3+)
- **T06** (programming): Light - secure coding (G5+)
- **T35** (impacts): Strong - security implications (G7-8)
- **T36** (ethics): CRITICAL - responsible security practices (G7-8)

**Capstone Skills (5)**: LARGEST GROUP
- T32.G8.01-05: Advanced security concepts with ethical dimensions
- This topic has the most capstone skills because security is both technical AND ethical

### T33: Platforms, APIs & External Services (29 skills)

**What Students Learn**:
- Apps vs websites vs tools
- Platforms as managed spaces (App Store, Google Play, etc.)
- APIs for accessing external services
- Using AI services and external data
- Ethical implications of API dependencies

**Grade Progression**:
- **G1**: What are apps, websites, games?
- **G2**: Tools and their purposes
- **G3**: Understanding platforms
- **G4**: How platforms work (app stores, updates)
- **G5**: Calling external services, basic API concepts
- **G6**: Complex API usage, data access
- **G7**: AI services as APIs (vision, NLP, etc.)
- **G8**: Ethical considerations with external dependencies

**Key Dependencies**:
- Within-topic: Sequential
- **T06** (programming): CRITICAL - APIs require programming (26 connections)
  - G3-4: Understanding without coding
  - G5+: Actually using APIs requires T06.G5-6 (functions, advanced concepts)
- **T11** (functions): Medium - functions as API concepts
- **T22-T23** (AI): Strong - AI APIs (G7-8)
- **T36** (ethics): Light - ethical API use (G8)

**Capstone Skills (4)**:
- T33.G8.01-04: Building systems with external services

**NOTE**: T33 is the most programming-dependent of T30-T36 topics

- **T21-T24** (AI): Strong - understanding AI impacts (G5-6)

**NOTE**: T35 is CRUCIAL bridge between applications and ethics

## Implementation Pathways

### For Teachers: Grade-by-Grade Integration

#### Primary (K-2): Build Awareness
- Introduce each topic's foundational concepts
- Emphasize diverse perspectives and inclusion
- Connect to students' own technology use

#### Upper Elementary (3-4): Deepen Understanding
- Explore how technology works in their lives
- Discuss both positive and negative impacts
- Begin understanding design choices

#### Middle School (5-6): Technical Integration
- Combine T30-T36 with T06-T13 programming skills
- Project-based learning (build with APIs, consider impacts)
- Begin recognizing systemic implications

#### Late Middle School (7-8): Synthesis & Responsibility
- Complete "Responsible Computing Pathway"
- Capstone projects integrating technical + ethical
- Prepare for high school CS with ethical awareness

1. Understand personal security needs
2. Recognize impacts on their own lives
3. Make informed decisions about tech use

1. Know how devices and networks work
2. Understand platforms and APIs
3. Use technology mindfully

#### Responsible Creators (T36 + T35)
1. Build ethically-conscious projects
2. Collaborate inclusively
3. Consider societal impacts of their work

#### Historically-Informed Problem Solvers (T34 + T35)
1. Learn from computing history
2. Recognize patterns and consequences
3. Approach future challenges thoughtfully

## Key Findings & Recommendations

### Findings

1. **Strong Ethical Foundation**
   - T32 → T35 → T36 creates natural responsibility arc
   - Security framed as both technical AND ethical
   - Ethics not afterthought but core thread

2. **Good Programming Integration**
   - T33 appropriately requires T06 foundation
   - Other topics intentionally lighter on coding
   - Allows access to systems thinking without deep programming

3. **AI Ethics Well-Placed**
   - Connected throughout (T21-T24 references)
   - Impacts understanding (T35) connects to ethics (T36)
   - Bias analysis bridges (T28.G7.03 + T35.G7.01)

4. **History Appropriately Integrated**
   - T34 doesn't stand alone
   - Connects to design thinking (T05) and impacts (T35)
   - Grade 7-8 use history to inform current decisions

5. **Capstone Skills Well-Marked**
   - 29 capstones appropriately distributed
   - T36.G8.01 is true K-8 culmination
   - Each topic has 4-5 capstone skills

### Recommendations

1. **Strengthen T34 Connections**
   - Add explicit links to modern computing challenges
   - Use history to contextualize current ethics issues
   - Examples: Historical privacy, surveillance, AI

2. **Enhance T30 Practicality**
   - Add more hands-on device exploration
   - Light IoT/sensor activities for G5-6
   - Connect to T06 programming (light)

3. **Ensure T33 Prerequisites**
   - Verify students have T06.G5-G6 before T33.G5
   - Functions (T11) should be prerequisite for API use
   - Create bridge projects combining T06 + T33

4. **Emphasize T32-T35-T36 Pathway**
   - Explicitly teach this as integrated "ethics arc"
   - Projects spanning all three topics
   - Connect to real-world security/impact/responsibility issues

5. **Create Cross-Curricular Opportunities**
   - T34 with history/social studies
   - T35 with health (screen time, mental health)
   - T32 with math (encryption)
   - T36 with SEL (decision-making, teamwork)

## Conclusion: The Role of T30-T36 in K-8 CS Education

### The Complete Framework

**K-8 Computer Science Education = Technical Skills + Systems Thinking + Social Responsibility**

| Domain | Focus | Skills |
|--------|-------|--------|
| T01-T05 | **Thinking** | Algorithms, design, problem-solving |
| T06-T13 | **Programming** | Core language and logic skills |
| T14-T24 | **Applications** | Creating with technology |
| T25-T29 | **Data** | Quantitative literacy |
| **T30-T36** | **Systems & Society** | How it all fits together + responsibility |

### T30-T36 Specific Role

T30-T36 transforms students from **tool users** into **informed technologists**:

- **T30-T33**: Understanding the computing ecosystem (hardware, networks, platforms, security)
- **T34**: Learning from computing history
- **T35**: Understanding societal impacts
- **T36**: Acting with responsibility and ethics

### Ultimate Outcome

Students completing K-8 CS with T30-T36:

1. Can build meaningful projects (T06-T13, T14-T24)
2. Understand data literacy (T25-T29)
3. Think algorithmically (T01-T05)
4. **Understand how computing shapes society (T30-T36)**
5. **Make ethical decisions about technology (T32-T36)**
6. **Consider impacts of their work (T35-T36)**
7. **See computing as professional responsibility (T36)**

This is computational citizenship: technical competence combined with ethical awareness.
