CHART TYPE SPECIFICATIONS
═══════════════════════════════════════════════════════════════════════════════

Line Chart:
  • Best for: Trends, time series, continuous data
  • Data source: List or table column(s)
  • Multi-series: Yes
  • Categories: Numeric indices
  • Configuration: Position (X,Y), Size (W,H)

Bar Chart:
  • Best for: Comparisons, categorical data
  • Data source: List or table column(s)
  • Multi-series: Yes
  • Categories: Numeric indices
  • Configuration: Position (X,Y), Size (W,H)

Pie Chart (Simple):
  • Best for: Composition, parts of whole
  • Data source: List or table column(s)
  • Multi-series: No
  • Categories: Numeric indices
  • Configuration: Position (X,Y), Size (W,H)

Pie Chart (Categorical):
  • Best for: Labeled composition, category breakdown
  • Data source: Table with category + value columns
  • Multi-series: No
  • Categories: Column values
  • Configuration: Position (X,Y), Size (W,H)

Percentage Chart:
  • Best for: Proportional representation
  • Data source: List or table column(s)
  • Multi-series: No
  • Categories: Numeric indices
  • Configuration: Position (X,Y), Size (W,H)

Progress Bar:
  • Best for: Progress tracking, completion status
  • Data source: Numeric value
  • Input: Current value and Total value
  • Features: Custom colors, border styling, naming
  • Configuration: Position (X,Y), Size (W,H), Colors

TYPICAL ANALYSIS WORKFLOW
═══════════════════════════════════════════════════════════════════════════════

Phase 1: DATA ACQUISITION
  ├─ Import CSV file
  │  └─ import file into table [raw_data v]
  │
  ├─ OR Read from Google Sheets
  │  └─ read from google sheet: url [...] range [A1:D100] into table [raw_data v]
  │
  └─ OR Query cloud database
     └─ fetch from collection [data v] into table [raw_data v] where <CONDITION>

Phase 2: DATA CLEANING & PREPARATION
  ├─ Delete invalid rows
  │  └─ delete rows with column [score] of value [0]
  │
  ├─ Delete missing data
  │  └─ delete rows with column [age] of value []
  │
  ├─ Transform data
  │  └─ copy list [ages v] to column [age] of table [data v]
  │
  └─ Sort data
     └─ sort table [data v] by column [name] [small to large v]

Phase 3: DATA ANALYSIS
  ├─ Compute statistics
  │  ├─ set [avg v] to [average v] of column [score]
  │  ├─ set [min v] to [smallest v] of column [score]
  │  └─ set [max v] to [largest v] of column [score]
  │
  ├─ GROUP BY analysis
  │  └─ set table [summary v] to [average v] of column [score] by column [category]
  │
  ├─ Pivot analysis
  │  └─ pivot [data v] into [crosstab v] row groups [gender,grade] columns [score] methods [average]
  │
  └─ Search/filter
     └─ set [found_row v] to row # of [John] in column [name]

Phase 4: VISUALIZATION
  ├─ Create charts
  │  ├─ draw [line v] chart using columns [score] from table [data v] ...
  │  ├─ draw [bar v] chart using columns [score] from table [data v] ...
  │  └─ draw pie chart using category [name] and value [score] from table [data v] ...
  │
  ├─ Show tables
  │  ├─ show table [data v]
  │  └─ show snapshot of table [summary v] from row (1) to (20) ...
  │
  └─ Create indicators
     └─ add progress bar as (50) out of total (100) ... as [status]

Phase 5: REPORTING & EXPORT
  ├─ Export to CSV
  │  └─ export table [summary v] as [report]
  │
  ├─ Push to Google Sheets
  │  └─ write into google sheet: url [...] starting cell [A1] from table [summary v]
  │
  ├─ Save to cloud
  │  └─ save table [summary v] to server as [analysis_results]
  │
  └─ Share results
     └─ Display snapshots, create charts, export files
