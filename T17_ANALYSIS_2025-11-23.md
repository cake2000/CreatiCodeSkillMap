## PART 3: SKILLS THAT ARE TOO BROAD

### 1. G6.05 - "Set up static collision groups for filtering" ‚ö†Ô∏è
**Current Coverage:** Combines 4 different operations
**Blocks involved:**
- `add to collision group [G]` (membership)
- `enable collision with group [G]` (filtering)
- `disable collision with group [G]` (filtering)
- Understanding the relationship between membership and filtering

**Issue:** Group MEMBERSHIP vs group FILTERING are conceptually distinct

**Recommendation:** Break into two skills:
```
G6.05 - Assign sprites to collision groups
Description: Students use `add to collision group [0-15]` to assign sprites to groups. They understand that each sprite can belong to multiple groups and that group membership is separate from collision filtering.

G6.05.A - Configure collision group filtering
Description: Students use `enable collision with group [G]` and `disable collision with group [G]` to control which groups their sprite collides with. They understand bidirectional filtering: both sprites must enable collision with each other's groups.

G6.05.01 - Dynamically modify collision groups at runtime (KEEP)
G6.05.02 - Use dominance groups for one-way pushing (KEEP)
```

### 2. G7.04.01 - "Use rotation speed and torque" ‚ö†Ô∏è
**Current Coverage:** Combines 3 different rotation paradigms
**Blocks involved:**
- `set rotation speed (SPEED)` - direct control
- `add torque (TORQUE)` - continuous force
- `apply torque impulse (TORQUE)` - instant force

**Issue:** Three fundamentally different approaches to rotation
- Direct = teleportation (kinematic)
- Torque = motor (physics-based continuous)
- Impulse = kick (physics-based instant)

**Recommendation:** Break into two skills:
```
G7.04.01 - Set rotation speed directly for spinning objects
Description: Students use `set rotation speed [value]` to directly control how fast a physics body spins (degrees/second). They use this for constant-speed rotating platforms, spinning coins, or turrets that need precise rotation control independent of forces.

G7.04.01.01 - Apply torque for physics-based rotation
Description: Students use `add torque [value]` for continuous rotational force and `apply torque impulse [value]` for instant spin. They compare continuous torque (like a motor) vs impulse torque (like a kick) and understand how torque interacts with angular damping and collisions.
```

## APPENDIX B: PEDAGOGICAL INSIGHTS

### Why Manual Physics First Works
The G5.02-G5.04 manual physics track before G5.05+ engine track is pedagogically brilliant:

**Benefits:**
1. **Conceptual Understanding:** Students learn WHAT gravity/velocity/friction ARE
2. **Debugging Skills:** Understanding internals helps debug engine behavior
3. **Transfer Learning:** Manual skills apply to non-physics motion (UI, particles, scrolling)
4. **Appreciation:** Students appreciate engine after manual implementation
5. **Tool Choice:** G5.12 teaches when to use each approach

**This matches professional practice:**
- Game dev tutorials teach manual before physics engines
- Physics courses teach equations before simulators
- Scratch community teaches manual motion before extensions

### Why Cross-Topic Dependencies Are Acceptable
Physics fundamentally requires:
- Loops (for continuous simulation)
- Conditionals (for collision response)
- Variables (for properties and state)

**Alternatives considered:**
1. Push physics to G9+ when loops/conditionals/variables mature ‚Üí **Loses K-8 engagement**
2. Create physics without programming ‚Üí **Not realistic**
3. Accept dependencies and co-teach ‚Üí **CHOSEN, practical**

**Real-world justification:**
- No game development curriculum separates physics from programming
- Teachers introducing physics naturally review foundational programming
- Student motivation from physics drives programming skill acquisition
