## Legend

- **★ SPLIT**: Skill was divided into multiple focused sub-skills
- **★ EXPANDED**: Series was expanded with additional skills
- **★ NEW**: Entirely new skill or series added
- **(number)**: Count of skills in that grade/series

## Skills by Type

### Variable Operations (G3)
```
Create → Set → Display → Use in Types (Number/Text/Boolean)
```

### List Operations (G3-G5)
```
Basic:     Create → Add → Display → Delete → Insert → Replace
Advanced:  Join → Split → Find → Contains
Stats:     Min/Max → Sum/Average → Median
Manipulate: Reverse → Reshuffle → Sort ↑/↓ → Copy/Append
```

### Table Operations (G3-G6)
```
Basic:      Create → Add Row → Display → Access Cell
Rows:       Count → Get Row → Delete Row → Delete All
Columns:    Add Column → Delete Column → Get Column
Advanced:   Insert Row → Replace Row → Replace/Change/Reduce Cell
Query:      Find Row → Lookup → Filter → Sort → Reshuffle
Transform:  Group By → Pivot → Copy/Append
```

### Persistence Operations (G6-G7)
```
Simple:     Save Value → Load Value (with keys)
CSV:        Export Table → Save Text → Load Text → Import Table
Local:      Save Table Direct → Load Table Direct
Database:   Insert → Fetch → Query → Update → Delete
Sheets:     Connect → Import → Export → Append → Update Cells
```

### AI-Data Integration (G8)
```
Vision:     Face Detection → Body/Hand Pose → Store in Tables
Language:   NLP Analysis → Store Results → Semantic Search
ML:         Prepare Training Data → Train Models → Store Predictions
```

## Skill Progression Patterns

### Pattern 1: Foundation → Application → Integration
```
Example: Variables
G3.00.01.01: Create variable (foundation)
G3.02.01:    Use number variables for scoring (application)
G5.01.02.01: Define game state variables (integration)
```

### Pattern 2: Simple → Filtered → Aggregated
```
Example: Table Queries
G5.06.02:    Query by single value (simple)
G6.05.01.03: Filter with compound conditions (filtered)
G6.05.02:    Aggregate using sum/avg/median (aggregated)
```

### Pattern 3: Local → Persistent → Shared
```
Example: Data Storage
G3.00.01.02: Set variable value (local, session-only)
G6.06.01.01: Save to server storage (persistent, private)
G7.05.01.02: Insert to database collection (persistent, shared)
```

### Pattern 4: Understand → Implement → Optimize
```
Example: Data Structures
G4.01:       Build schema diagrams (understand)
G5.06.01:    Create multi-column tables (implement)
G6.08.02:    Group by column values (optimize)
```
