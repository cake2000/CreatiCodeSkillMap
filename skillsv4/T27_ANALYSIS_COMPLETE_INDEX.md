## ğŸ’¬ COMMON QUESTIONS

### Q: Why are there so many new skills to add?
**A:** T27 had scaffolding gaps, especially in G3-G5 where concepts jumped too quickly. Adding 7 skills (spread across 4 grades) fills these gaps and creates smoother progression.

### Q: Can I skip the block verification?
**A:** NO. Block verification is CRITICAL. Many skills reference specific CreatiCode blocks. If blocks don't exist, skills need major rewrites or may be impossible to implement.

### Q: Which issues should I fix first?
**A:** Follow this order:
1. Verify blocks (foundation for everything else)
2. Fix dependency violations (affects numbering)
3. Split over-broad skills (core fixes)
4. Add new skills (fills gaps)
5. Polish descriptions (final refinement)

### Q: How does this compare to other topics?
**A:** T27 is similar to other topics in needing block verification and scaffolding improvements. It's better than average in having a strong K-2 foundation. The widget gap is unique to interactive data topics.

### Q: What if CreatiCode doesn't have pivot tables or Google Sheets blocks?
**A:** Three options:
1. Implement algorithmic versions (loops + conditionals)
2. Mark skills as "advanced/optional"
3. Move skills to later grade when implementation is simpler
The analysis provides fallback plans for each questionable block.

### Q: Will these changes break other topics?
**A:** Unlikely. Analysis preserves all cross-topic dependencies (T01, T06-T10, T24-T26). Only intra-T27 dependencies are modified. However, search for T27.G4.04b before renumbering it.
