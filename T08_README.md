## Common Questions

### Q: Why was T08.G3.03b added?
**A:** Students were expected to trace if/else code (G3.04) before building it. The new skill fills this scaffolding gap, following "build → trace → debug" progression.

### Q: Why does T08.G4.02 now depend on T08.G4.01?
**A:** To ensure students learn AND before OR. This creates proper pedagogical sequencing for boolean operators.

### Q: Why were redundant dependencies removed?
**A:** When skill A depends on B, and B depends on C, then A doesn't need to explicitly depend on C. Removing redundancies simplifies the dependency graph without losing any constraints.

### Q: Are cross-topic dependencies preserved?
**A:** Yes! All dependencies to T06 (Events), T07 (Loops), T09 (Variables), T04 (Patterns), and T03 (Decomposition) are intentionally preserved as per Phase 1 instructions.

### Q: How do I know the changes are correct?
**A:** Use T08_VERIFICATION_CHECKLIST.md which has 100+ validation checks covering structure, content, dependencies, and cross-topic integration.

### Q: What if I find an issue after implementation?
**A:** See "Rollback Procedure" in T08_VERIFICATION_CHECKLIST.md. Always keep your backup file.
