## Redesign Philosophy

### Core Principles

1. **Pictures First, Words Support**
   - All activities use pictures as primary content
   - Text limited to 5-8 words (K), 8-12 words (G1), 12-15 words (G2)
   - Audio narration for all instructions

2. **No Coding Required**
   - K-2 students interact through clicking, dragging, sorting
   - Concepts taught through concrete, visual examples
   - Grade 2 includes "bridge activities" that introduce coding vocabulary through pictures

3. **Developmentally Appropriate**
   - Kindergarten: 2-3 minute activities, 2-3 choices, very concrete
   - Grade 1: 3-4 minute activities, 3-5 choices, familiar contexts
   - Grade 2: 4-5 minute activities, 4-6 choices, bridges to abstract concepts

4. **Auto-Gradable**
   - Clear correct answers
   - Immediate feedback
   - Multiple attempts with hints
   - No subjective grading

5. **Accessible to All**
   - WCAG 2.1 AA compliant
   - Audio support
   - Keyboard navigable
   - Large touch targets
   - Color-blind safe

## Detailed Topic Analysis

### T01: Everyday Algorithms & Step-by-Step Thinking

**Original Skills (G1-G2):**
- G1.01: Write or draw steps for a simple task
- G1.02: Trace and predict the outcome of a simple algorithm
- G1.03: Identify missing or incorrect steps in an algorithm
- G1.04: Compare two algorithms for the same task
- G2.01: Write an algorithm with "if/then" choice
- G2.02: Write an algorithm with a repeat statement
- G2.03: Trace an algorithm with branching or repetition
- G2.04: Unplugged maze or robot navigation with instructions

**Redesigned Skills:**

**Kindergarten (NEW):**
1. Help the Puppy Get Ready for Bed (3-step bedtime sequence)
2. Making a Sandwich: What's the Right Order? (3-step food prep)
3. Getting Dressed: Morning Routine (3-step logical sequence)
4. Washing Hands: Do It Right! (3-step health routine)

**Grade 1 (Redesigned):**
1. Plant a Seed: Steps in Order (4-5 step sequence with nature theme)
2. What Happens Next? Story Sequence (predict next step in visual story)
3. Find the Missing Step (identify missing step in sequence)
4. Two Ways to Do It: Which is Better? (compare two visual sequences)

**Grade 2 (Redesigned with Bridges):**
1. Choose the Path: If-Then Decisions (conditional logic through path choices) ← **Bridges to if-then**
2. Patterns that Repeat: Can You See It? (identify repeating units) ← **Bridges to loops**
3. Follow the Steps: What Does It Make? (trace sequence to predict outcome)
4. Guide the Robot Through the Maze (interactive navigation with directional commands) ← **Bridges to sequences**

**Key Changes:**
- Removed text-heavy "write algorithm" tasks
- Added concrete, visual sequencing with familiar contexts
- Grade 2 introduces coding vocabulary through visual tasks
- All activities use drag-drop or click interactions

**Rationale:**
- Young children understand algorithms best through concrete, familiar routines
- Sequencing pictures builds logical thinking without requiring coding syntax
- Grade 2 activities prepare students for block-based coding in Grade 3+

### T04: Patterns & Reusable Solutions

**Original Skills (G1-G2):**
- G1.01: Find repeated blocks in a script
- G1.02: Match a repeated picture story to code
- G1.03: Recognize a template structure in sprite code
- G1.04: Build a simple repeating script (no loops)
- G2.01: Identify repeating sections in longer scripts
- G2.02: Replace repeated code with a loop
- G2.03: Use a template project and customize it
- G2.04: Predict what a simple repeated pattern produces

**Redesigned Skills:**

**Kindergarten (NEW):**
1. Animal Pattern: What Comes Next? (AB pattern with animals)
2. Color Pattern: Red, Blue, Red, Blue (AB pattern with colors)
3. Shape Pattern: Circle, Square, Circle (AB pattern with shapes)
4. Big-Small Pattern (AB pattern with size attribute)

**Grade 1 (Redesigned):**
1. Find the Repeating Part (identify unit in ABC pattern)
2. Story Pattern Match (match repeating stories to patterns)
3. Template Time: Which Ones Are the Same? (identify same template)
4. Make Your Own Pattern (create repeating pattern)

**Grade 2 (Redesigned with Bridges):**
1. Find All the Patterns (identify multiple repeating sections)
2. Short-Cut the Repeat: Circle It Once (identify what to repeat instead of showing multiple times) ← **Bridges to loop concept**
3. Customize the Template (modify template while keeping structure) ← **Bridges to template/function concept**
4. Pattern Prediction: What Will It Make? (predict result of repeating pattern)

**Key Changes:**
- Removed code-based pattern recognition
- Added visual pattern activities (colors, shapes, objects)
- Grade 2 explicitly connects repeating patterns to loop concept through visual "shortcuts"
- Progression from simple AB patterns (K) to complex patterns and templates (G2)

**Rationale:**
- Pattern recognition is fundamental to understanding loops and reusability
- Visual patterns are concrete and accessible to young children
- Grade 2 activities explicitly name coding concepts while maintaining picture-based interaction

### T25: Data Representation

**Original Skills (G1-G2):**
- G1.01: Collect and sort data by simple categories
- G1.02: Compare numbers as "more," "less," or "same"
- G1.03: Recognize that numbers have special meaning
- G1.04: Create a simple tally or picture chart

**Redesigned Skills:**

**Kindergarten (NEW):**
1. Sort by Color (single attribute sorting)
2. Big or Small? (size classification)
3. What Kind Is It? (type categorization: animals/toys/food)

**Grade 1 (Redesigned):**
1. Sort by Two Things (multi-attribute classification)
2. More, Less, or Same? (visual comparison)
3. Numbers Tell Us Things (recognize numeric data)
4. Make a Picture Chart (create simple chart from data)

**Grade 2 (Redesigned with Bridges):**
1. Number or Not? (classify data types) ← **Bridges to data types**

**Key Changes:**
- Removed abstract variable creation tasks
- Added concrete sorting and classification activities
- Grade 2 introduces "variable" vocabulary through visual "boxes" that store values
- Clear progression from single attribute (K) to multi-attribute (G1) to variable concept (G2)

**Rationale:**
- Data concepts are best learned through tangible sorting and organizing
- "Variable boxes" provide concrete metaphor for abstract variable concept
- Grade 2 explicitly teaches coding vocabulary while maintaining picture-based interaction

## Progression Within Each Topic

### Example: T01 (Algorithms) Progression

**Kindergarten:**
- **Concept:** Steps happen in order
- **Complexity:** 3 steps
- **Context:** Familiar routines (bedtime, getting dressed)
- **Interaction:** Drag 3 pictures to 3 slots
- **Time:** 2 minutes

**Grade 1:**
- **Concept:** Steps in order + debugging (finding errors)
- **Complexity:** 4-5 steps
- **Context:** School/home activities (planting seeds, cooking)
- **Interaction:** Drag 5 items, predict next step, compare sequences
- **Time:** 3-4 minutes

**Grade 2:**
- **Concept:** Sequences with choices (if-then) + repetition
- **Complexity:** 6-8 steps with branches
- **Context:** Problem-solving (robot navigation, maze solving)
- **Interaction:** Path tracing, conditional choices, prediction
- **Time:** 4-5 minutes
- **Bridge:** Introduces "if-then" and "repeat" vocabulary through visual tasks

## Dependencies

### Minimal Dependencies for K-2

**Design Decision:** K-2 skills have minimal dependencies to allow flexible learning paths.

**Kindergarten:** No dependencies (all skills accessible from start)

**Grade 1:**
- May reference Kindergarten concepts but not required
- Example: G1 pattern skills build on K patterns, but K is not prerequisite

**Grade 2:**
- May reference G1 concepts
- Some skills build on previous skills within same topic
- Example: T25.G2.02 (Make a Variable Box) could follow T25.G2.01 (Number or Not?)

**Rationale:**
- Young children benefit from spiral curriculum (revisiting concepts)
- Teachers need flexibility to sequence based on student readiness
- Skills designed to be self-contained with scaffolding

## Activity Type Choices: Rationale

### drag_drop_sequence (34 skills)
**Why:** Sequencing is fundamental to algorithmic thinking. Drag-drop is intuitive for young children and works on touch screens.
**Best for:** T01 (algorithms), T04 (patterns), T25 (data organization)
**Age appropriateness:** K: 3 items, G1: 4-5 items, G2: 6-8 items

### sort_categories (28 skills)
**Why:** Classification builds data organization and logical thinking. Sorting into bins is concrete metaphor.
**Best for:** T25 (data representation), T26 (data collection), T32 (cybersecurity), T35 (impacts)
**Age appropriateness:** K: 2 bins, G1: 2-3 bins, G2: 3-4 bins

### click_select (26 skills)
**Why:** Simplest interaction. Good for identification, choice, and comparison tasks.
**Best for:** All topics, especially T30 (hardware), T32 (safety), T34 (history)
**Age appropriateness:** K: 2-3 choices, G1: 3-4 choices, G2: 4-6 choices

### pattern_complete (8 skills)
**Why:** Specifically designed for pattern recognition. Focused on T04.
**Best for:** T04 (patterns)
**Age appropriateness:** K: AB patterns, G1: ABC patterns, G2: Complex patterns

### match_pairs (8 skills)
**Why:** Good for connecting related concepts (device to use, old to new, inventor to invention).
**Best for:** T30 (hardware), T34 (history)
**Age appropriateness:** K: 3-4 pairs, G1: 4-5 pairs, G2: 5-6 pairs

### counting_interaction (3 skills)
**Why:** Essential for data collection and analysis.
**Best for:** T26 (data collection), T27 (data analysis)
**Age appropriateness:** K: 1-5 items, G1: 1-10 items, G2: 3-12 items

### Other Activity Types
**yes_no_sort:** Used for binary classification (safe/unsafe, private/not private)
**hot_spot_click:** Used for identifying parts in images (computer parts, devices in scene)
**multiple_choice_visual:** Used for single-choice questions with picture answers
**path_tracing:** Used for navigation tasks (robot mazes, algorithm paths)

## How These Skills Set Foundation for Grade 3+ Coding

### Conceptual Bridges

**T01: Algorithms**
- K-2: Sequencing pictures of daily routines
- **→ Grade 3+:** Sequencing code blocks to create programs

**T04: Patterns**
- K-2: Identifying repeating visual patterns
- **→ Grade 3+:** Using loop blocks to repeat code

**T25: Data Representation**
- K-2: "Variable boxes" that store values
- **→ Grade 3+:** Creating and using variables in code

**T26-T27: Data Collection & Analysis**
- K-2: Organizing data into tables, creating charts
- **→ Grade 3+:** Using lists and data structures in code

**T30: Hardware**
- K-2: Input → Processing → Output flow
- **→ Grade 3+:** Understanding how programs process input and produce output

**T32: Cybersecurity**
- K-2: Privacy concepts, safe behaviors
- **→ Grade 3+:** Safe coding practices, protecting user data

**T34: History**
- K-2: Technology evolution, diverse inventors
- **→ Grade 3+:** Understanding computing in context, diverse contributions

**T35: Impacts**
- K-2: Good and bad effects of technology
- **→ Grade 3+:** Responsible computing, considering impact of programs created

### Vocabulary Bridges (Grade 2)

Grade 2 skills explicitly introduce coding vocabulary through picture-based tasks:

- **"Variable"** - introduced through "variable boxes" that store values
- **"If-then"** - introduced through path choice scenarios
- **"Repeat/Loop"** - introduced through identifying repeating patterns
- **"Input/Output"** - introduced through device interaction
- **"Template"** - introduced through customizable patterns
- **"Algorithm"** - used throughout to describe step-by-step sequences

### Skill Progression

**Kindergarten:** Build foundation
- Sequencing, pattern recognition, sorting, comparing

**Grade 1:** Expand understanding
- More complex sequences, pattern creation, data organization

**Grade 2:** Bridge to coding
- Introduce vocabulary, show connections, prepare for block-based coding

**Grade 3+:** Apply in coding
- Use CreatiCode block-based coding to create programs applying all concepts
