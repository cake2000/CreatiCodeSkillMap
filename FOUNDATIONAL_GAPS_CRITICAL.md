# Critical Foundational Gaps - Immediate Action Required

**Date:** 2025-11-17
**Severity:** CRITICAL - BLOCKING
**Status:** No coding instruction possible without these

---

## Executive Summary

**The CreatiCode K-8 Skill Map has ZERO coding skills for grades 3-8.**

This document identifies the most critical foundational block skills that MUST be created immediately to enable any coding instruction.

---

## Critical Gap #1: Zero Grade 3 Skills (SEVERITY: BLOCKING)

### The Problem

Grade 3 is defined in the spec as "THE BRIDGE" from picture-based to block coding, with 5 critical gateway skills. Currently:

- **Expected:** 60-83 Grade 3 skills
- **Actual:** 0 skills
- **Gap:** 100% of Grade 3 curriculum missing

### Impact

- Students complete Grade 2 with strong conceptual foundation
- Students enter Grade 3 expecting to start coding
- **NO coding curriculum exists**
- Students cannot progress to grades 4-8
- Entire skill map is blocked

### Immediate Action Required

Create **Minimal Viable Grade 3 Curriculum (MVC)**: 25 skills across 5 block categories.

## Critical Gap #2: No Event Blocks (SEVERITY: CRITICAL)

### The Problem

**Students cannot start any program without event blocks.**

Currently:
- K-2: 11 conceptual skills about cause-effect, triggers
- G3-8: 0 skills teaching how to use event blocks
- **Result:** Students understand the concept but cannot write code

### Foundational Blocks Completely Missing

| Block | Why Critical | When Needed |
|-------|--------------|-------------|
| `when green flag clicked` | THE entry point for all programs | First coding skill ever |
| `when this sprite clicked` | Second most common event | First week of Grade 3 |
| `when key pressed` | Required for games/interaction | First month of Grade 3 |

### Impact Without These Blocks

- **Cannot start programs** - Students cannot make anything run
- **Cannot create games** - No way to control with keyboard
- **Cannot make interactive projects** - No way to respond to user
- **Gateway skill blocked** - T06.G3.01 (190 dependent skills) cannot exist

### Immediate Action Required

Create these 5 skills FIRST (before anything else):

1. **T06.G3.01**: Start program with green flag (GATEWAY SKILL)
2. **T06.G3.02**: Make sprite respond to clicks
3. **T06.G3.03**: Use multiple event handlers in one project
4. **T06.G3.04**: Respond to arrow key presses
5. **T06.G3.05**: Respond to space bar press

**Timeline:** Week 1, Days 1-2

## Critical Gap #4: No Motion Blocks (SEVERITY: HIGH)

### The Problem

**Students cannot move sprites, which is the most expected/desired behavior in Scratch-like environments.**

Currently:
- K-2: 8 conceptual skills (direction, paths, position)
- G3-8: 0 skills teaching how to use motion blocks
- **Result:** Minimal concept foundation, no block practice

### Foundational Blocks Completely Missing

| Block | Why Critical | When Needed |
|-------|--------------|-------------|
| `move steps` | Most basic sprite motion | First coding project |
| `turn degrees` | Change direction | First coding project |
| `go to x:y` | Positioning | Week 1 |
| `if on edge, bounce` | Keep sprites on screen | Week 2 |

### Impact Without These Blocks

- **No visible output** - Students want to see sprites move
- **Projects are static** - Boring, low engagement
- **Cannot create games** - All games need motion
- **Student frustration** - "Why can't I make it move?"

### Immediate Action Required

Create these 7 skills in Week 1-2:

1. Move sprite forward with move block
2. Turn sprite with turn block
3. Move in different directions (up/down/left/right)
4. Make sprite return to starting position
5. Create circular motion with repeat + turn
6. Bounce off edges
7. Use glide for smooth animation

**Timeline:** Week 2, Days 1-3

- **Teacher dependency** - Cannot self-correct
- **Learning slows** - Cannot identify own mistakes
- **Gateway skill blocked** - T13.G3.01 cannot exist

### Immediate Action Required

Create these 5 skills in Week 3:

1. **T13.G3.01**: Use say blocks to debug (GATEWAY SKILL)
2. Test code one block at a time
3. Identify when code doesn't run

5. Use XO assistant to get help

## Summary: The Minimum Viable Grade 3 Curriculum (MVC)

To enable ANY coding instruction, these 41 skills must be created:

### Week 1: Basic Operations (20 skills)

**Events (5 skills) - Days 1-2:**
- when green flag clicked
- when sprite clicked
- multiple event handlers
- arrow key presses
- space bar press

**Control - Basics (4 skills) - Days 3-4:**
- wait for timing
- repeat for patterns
- forever for continuous motion
- repeat + motion

**Control - Conditionals (4 skills) - Days 4-5:**
- if-then for conditions
- if-then-else for two paths
- comparison operators
- if-touching for collision

**Operators (5 skills) - Integrated throughout:**
- addition and subtraction
- multiplication and division
- pick random
- comparison operators
- combining operators

**Motion (2 skills) - Day 5:**
- move sprite forward
- turn sprite

### Week 2: Visual Feedback & Interaction (13 skills)

**Motion continued (5 skills) - Days 1-2:**
- move in 4 directions
- return to starting position
- circular motion
- bounce off edges
- glide for smooth animation

**Looks (6 skills) - Days 3-5:**
- show and hide sprites
- say messages
- switch costumes
- next costume for animation
- change size
- pulsing effect

**Sensing (2 skills) - Day 5:**
- detect sprite touching sprite
- detect sprite touching edge

### Week 3: Data & Debugging (8 skills)

**Variables (5 skills) - Days 1-3:**
- create variable for score (GATEWAY)
- change variable to count
- display variable
- use variable in motion
- reset variable

**Debugging (3 skills) - Days 4-5:**
- use say blocks to debug (GATEWAY)
- test code incrementally
- fix common errors

### Week 4: Integration Projects (Optional)

Create 5-10 project-based skills that combine multiple block types:
- Simple catch game (motion + variables + collision)
- Interactive story (events + looks + say)
- Animation (looks + control + timing)
- Chase game (motion + sensing + control)
